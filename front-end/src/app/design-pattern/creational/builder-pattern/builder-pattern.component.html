Builder Pattern - Creational Pattern

Bản chất của Builder Pattern đó là xây dựng Object từng phần.

Nó được sử dụng khi ta muốn Build một Object quá phức tạp, mà việc init nó phải thực hiện theo từng bước, nhưng ta không biết/ không quan tâm tới thứ tự của từng bước đó.

Ngoài ra nó còn được sử dụng để xây dựng 1 Object mới bằng cách mở rộng một Object đã có sẵn

Builder Pattern dùng trong việc quản lý những đối tượng một cách phức tạp. Nếu như Abstract Factory tiếp cận theo kiểu có thực đơn trước, rồi tạo ra luôn sản phẩm, thì Builder Pattern tiếp cận theo hướng sử dụng step by step để đạt được mục đích cuối cùng.

Cùng trở lại cửa hàng góc phố thôi nào.

Trong trường hợp này, cái khai nguyên liệu - sản phẩm của anh phụ bếp - Abstract Factory kia tỏ ra vô dụng rồi. Tuy nhiên, anh đầu bếp vẫn có thể chế biến món ăn mà chị kia yêu cầu, nhờ vào cái miếng giấy anh phục vụ đưa.

Miếng giấy ấy là một ví dụ của Builder Pattern.

Introduction

Builder Pattern says that "construct a complex object from simple objects using step-by-step approach"

Builder Pattern sử dụng trong trường hợp object phức tạp và không theo một khuôn khổ nào. Việc truyền parameter vào constructor của Object không thực sự khả thi ở đây, khi mà các properties của nó thay đổi xoành xạch. Thay vì thế, nó hướng tới cách tiếp cận từng phần. Nó sử dụng các Inner class để thực hiện điều này.

Nó đặc biệt hữu dụng khi không thể create object với chỉ 1 step, hoặc dùng constructor, hoặc dùng 1 method nào đó.

Builder Pattern cũng được sử dụng khi ta muốn mở rộng 1 class đã có từ trước. Lúc này ta sẽ thêm các thuộc tính mới vào thông qua một Builder.

Builder giải quyết vấn đề mà Factory gặp phải, đó là khi Object có quá nhiều tham số, thì Factory sẽ trở nên vô dụng. Builder không sử dụng toàn bộ tham số để khởi tạo Object liền, thay vào đó nó truyền các tham số vào 1 cách từ từ, tiếp cận theo hướng xây dựng từng phần trước khi quyết định kết quả trả về là gì.

Xem ví dụ về cái ở trên kia, rõ ràng ta thấy việc sử dụng thực đơn là vô nghĩa trong trường hợp này, mà ta cần phải xây dựng từng phần - bằng danh sách liệt kê trong tờ giấy đấy. Kết quả trả về cuối cùng là món ăn của cô gái.

Implements

Trong Java, để tạo ra một Builder ta cần làm như sau:

  1. First of all you need to create a static nested class and then copy all the arguments from the outer class to the Builder class. We should follow the naming convention and if the class name is Computer then builder class should be named as ComputerBuilder.
  2. Java Builder class should have a public constructor with all the required attributes as parameters.
  3. Java Builder class should have methods to set the optional parameters and it should return the same Builder object after setting the optional attribute.
  4. The final step is to provide a build() method in the builder class that will return the Object needed by client program. For this we need to have a private constructor in the Class with Builder class as argument.
Advantages & Disadvantages
Advantages

Giúp ta phân chia rõ ràng giữa constructor và representation của Object

Cung cấp cách tốt hơn để kiểm soát trong quá trình xây dựng 1 object

Hỗ trợ thay đổi các representation bên trong của các đối tượng

Disadvantages

Có chăng cũng chỉ là thêm vài dòng code mà thôi.

Usage

Builder thường được dùng khi mà việc khởi tạo các Object phức tạp.

Khi ta muốn bổ sung thêm các builder method mà không cần quan tâm tới các object hiện tại - tăng khả năng mở rộng

Summary

Builder Pattern là một Pattern rất hữu dụng. Nó có tác dụng build một object step by step.

Mục đích sử dụng chính của nó đó là tạo nên những object phức tạp, theo từng bước, dựa vào tham số truyền vào.

Ngoài ra nó còn được sử dụng rất nhiều trong việc mở rộng dựa vào các object hiện tại.
