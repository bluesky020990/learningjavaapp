Overview

Trong bài học này ta cùng tìm hiểu về các module chính của Spring Framework.

Spring gồm 5 module chính:

  • Spring Core Container
  • AOP, Aspects, Instrumentation
  • Data Access/Integration
  • Web (MVC, Remote)
  • Và cuối cùng là Test
Spring Modules

Spring Framework bao gồm nhiều modules như core, beans, context, expression language, AOP, Aspect, Instrumentation, JDBC, ORM, OXM, JMS, transaction, Web, Servlet, Structs. Nó được gom lại thành từng group bao gồm Test, Core Container, AOP, Aspect, Instrumentation, Data Access/ Integration, Web/Remoting.

Dưới dây là sơ đồ cụ thể:

Test

Layer này support cho việc testing bằng cách sử dụng JUnit và TestNG

Spring Core Container

Container này bao gồm 3 phần, core, beans, context và Expression Language.

Core and Beans cung cấp các IOC và DI feature trong khi đó Context lại đóng vai trò hỗ trợ cho việc chuẩn hóa. Cuối cùng, Expression Language là một phần mở rộng cho EL được định nghĩa trong JSP. Nó cung cấp các hỗ trợ để thiếp lập và nhận các giá trị thuộc tính, gọi các phương thức, truy cập các collection và indexers, các variable, logical và số học, truy xuất các đối tượng theo tên,

AOP, Aspects and Instrumentation

Modules này hỗ trợ cho việc lập trình theo khía cạnh, nơi ta có thể đặt các Advices, Pointcuts, để tách mã.

Aspect Modules cung cấp các hỗ trợ để tích hợp với AspectJ

Trong khi đó instructment modules cung cấp hỗ trợ cho các class instrucmentation và class loader implementations

Data Access / Integration

Nhóm này bao gồm các modules như JDBC, ORM, OXM, JMS và transaction Modules. Các module này về cơ bản cuan cấp các hỗ trợ để tuong tác với database.

Web

Nhóm này bao gồm các moduels như Web, Web-Servlet, Web-Struts, và Web-Portlet. Các module này cung cấp các hỗ trợ để tạo nên các ứng dụng web.

Spring Components

Trong Spring, có 3 annotation thường được sử dụng nhất cho bean, đó là @Component, @Repository, và @Service.

Đối với Spring thì 3 cái này đều là bean, nó đều được scan theo class path nhưng tốt hơn hết, ta cần phân loại chúng theo đúng mục đích sử dụng.

Dưới đây ta sẽ đi chi tiết vào từng loại.

Component

ta có thể sử dụng @Component trên application để đánh dấu các bean là Spring's manager components.

Spring chỉ nhận và register bean với @Component mà ko tìm kiếm @Service hay @Repository nói chung.

Các bean này sẽ được register trong @Repository.

@Service và @Repository cũng là bean, nhưng chúng được dùng cho các mục đích khác, vì vậy chúng cần phải
