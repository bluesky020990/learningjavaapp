Angular Basic

Ở bài học này ta cùng đi vào Angular cơ bản.

Phần này bao gồm install Angular CLI, các components, service, directive trong Angular.

Nào cùng đi vào phần đầu tiên của bài học thôi.

Angular CLI
npm install -g @angular/cli

Nào, giờ ta cùng nhìn vào cấu trúc các file được generate ra nhé.


Mỗi file đã có những mô tả nên mình sẽ không đề cập lại nữa. Tuy nhiên trong số các bài học sau thì ta sẽ đi vào từng cái một. Tới lúc hiểu nó hơn thì ta có thể quay lại để update cho dễ hiểu.

app.,module.ts

Cùng tìm hiểu file này để xem cách thành phần của Angular nhé.

Trong file này gồm có 4 phần chính, phần declarations khai báo các component. Phần imports khai báo các module, phần providers cung cấp các service.

Như vậy, ta nhận thấy có 3 thành phần chính. Component, Module, và Service. Nào, giờ cùng đi vào chi tiết từng phần để tìm hiểu vai trò và trách nhiệm của nó nhé.

Component

Về mặt bản chất, Angular được cấu tạo từ các component. Mỗi component có nhiệm vụ render một vùng nào đó dựa trên data của nó (do init sẵn, do truyền vào, do sử dụng các service để get data).

Khi có sự thay đổi dữ liệu, thì nó sẽ tự động cập nhật lại cái cây DOM của nó.

Để tạo component ta sử dụng command line sau: (Lưu ý ta sẽ sử dụng các command line thay vì copy rồi sửa code nhé, vì nó vừa nhanh và đúng chuẩn của Angular. Chỉ thế thôi)

ng g c components/resource/angular-resources/AngularBasic

or

ng generate component _path_of_component/ComponentName

Sau khi chạy xong cái command trên ta có 4 file, Cùng xem file angular-basic.component.ts để hiểu rõ hơn nhé.

selector là tag name của component. Ta có thể mang nó đặt ở bất cứ đâu, chỉ cần bỏ vào <app-angular-basic></app-angular-basic> như thế này thì nó sẽ hiện.

templateUrl là nơi ta dùng chứa cái content HTML mà nó sinh ra. Các data sẽ được truyền vào ở đây. Về bản chất thì ta có thể xem nó là 1 view vậy.

styleURL là nơi chứa file css config. Lưu ý, ở chế độ mặc định, khi thêm class ở đây thì chắc chắn 1 điều nó chỉ tác động trong component này. Nếu mà muốn sử dụng global thì phải thêm các config khác. ví dụ ở đây ta thêm cái encapsulation và xem nó có cái gì nhé.

@Component({{'{' }}
    selector: 'app-angular-basic',
    templateUrl: './angular-basic.component.html',
    styleUrls: ['./angular-basic.component.scss'],
    encapsulation: ViewEncapsulation.None
})

encapsulation: ViewEncapsulation.None biến style trong file styleURL thành style global.

Vậy, ta nên sử dụng component khi nào.

Sử dụng khi nào hay còn gọi là mục đích sử dụng của component này là render dữ liệu. Nên nhớ mục đích của nó chỉ là render dữ liệu thôi nhé. Việc sử lý dữ liệu nên dành cho các service hoặc các class.

Vì sao lại vậy??

Bản chất của component đó là biến data thành view. Mục đích của nó là hiển thị view và nếu ta đưa function để process vào trong component này thì nó sẽ không thể reuse được ở các component khác. Vì vậy, Angular cung cấp cho ta 2 thứ, component để render view và Service để process dữ liệu. Khi làm thì nên lưu ý cái này nhé.

Tạo các variable trong Component như thế nào.

Thì tạo đơn giản thôi. Ví dụ: tạo 1 biến templateMessage và show ra màn hình.

{{templateMessage}}

Trong bài học này ta chỉ đi qua các tổng quan của nó. Riêng phần Angular View Function thì ta sẽ tìm hiểu trong bài tiếp theo.

Module

Module là các thư viện ngoài được embed vào trong app để có thể sử dụng. Nó tương tự như các pluggin. Khi sử dụng một cái Module thì cần tuân thủ các documentation của nó.

Trong Angular, có 2 cái Module là nổi tiếng nhất, đó là Material và Ng-Bootstrap.

Material thì bạn có thể tìm hiểu thêm ở đây.

Tương tự, ng-bootstrap thì có thể tìm hiểu thêm ở đây hoặc là xem qua các feature của nó ở các lesson sau.

Một số lưu ý khi sử dụng các Module

Trước khi sử dụng, thì các module phải được khai báo trong phần imports của app.module.ts.

Có thể tách riêng các Module ra để hạn chế dòng code import vào trong file app.module.ts cho nó khỏi rối mắt. Bạn có thể xem file src/app/material/bootstrap.module.ts hoặc src/app/material/material.module.ts để hiểu thêm về cách tách các module ra khỏi file import chính và import toàn bộ thông qua một module khác.

Khi sử dụng các Module từ bên ngoài, thì phải follow document của nó. Nên cẩn thận trong việc này vì có thể chỉ cần sai 1 chỗ nhỏ thôi thì cũng đủ để tìm ói máu.

Service

Service đúng như tên gọi của nó, nó là nơi lưu trữ các business logic, data để sử dụng cho các component.

Các tạo một service cũng tương tự như Component, ta sử dụng đoạn code sau:

ng g s services/BreadCrumb

Tại sao ta lại sử dụng service?

Như đã đề cập từ trước, service là nơi xử lý các business logic, get set các data cho các component.

Giả sử ta có component A và component B cùng tương tác với data C. Khi Component A đổi data C, nó chỉ đổi data của riêng bản thân nó và data mà B hiển thị vẫn như cũ. Khi một component get 1 data thông qua service, thì việc data của service bị thay đổi, cũng kéo theo sự thay đổi của các component sử dụng chúng. Xem file breadcrumb để biết thêm chi tiết.

Khi tạo 1 service, mặc định nó sẽ được set là injectable. Trong Angular, các service đều tuân thủ theo DI Pattern. providedIn: root cho ta biết service này sẽ được inject global.

Để sử dụng các service, ta đưa nó thông qua các component hoặc các service khác thông qua constructor.

Tương tự như Component, nó cũng cần được khai báo trong providers của app.module.ts.

Summary

Hiểu thêm về các thành phần của Angular giúp ta có thể sử dụng nó đúng vị trí ngay từ đầu, tránh được những rắc rối về sau.

Khuyến cáo nên sử dụng command để generate các Service và Component và ta sẽ chỉ cần tập trung vào phần implement nó mà thôi.

Luôn khai báo trước khi sủ dụng.

Nên tách các module thành những file riêng cho dễ quản lý.
